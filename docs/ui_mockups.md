- Tutor 回答时引用论文原文（RAG 证据来源）

- 面板区显示学习面板（计划进度 + 学习卡片），不再是纯 Brain 面板
- 流式输出：回答逐字/逐句出现（类似 ChatGPT）
- 上传 PDF 后首次交互，Tutor 直接分析文档（不再先问学习目标）

## 8. 引导流程自动化设计

### 8.1 状态机定义（v0.3.0 简化版）

> v0.3.0 将 6 阶段简化为 3 个 Tab，去掉了独立的 Input 和 Report 阶段

```
                    ┌──────────────────────┐
                    │       IDLE           │
                    │   (无 session)        │
                    └──────────┬───────────┘
                               │ 用户上传 PDF / 输入文字
                               │ → 创建 Session → 进入工作区
                               ▼
                    ┌──────────────────────┐
                    │    PLAN (规划 Tab)    │
                    │  PDF 处理 + 生成计划  │
                    └──────────┬───────────┘
                               │ 计划生成完毕
                               │ 用户点击「学习」Tab
                               ▼
                    ┌──────────────────────┐
                    │   STUDY (学习 Tab)    │◄──────┐
                    │  Tutor Agent 问答     │       │
                    └──────────┬───────────┘       │
                               │ 用户点击「测验」Tab │
                               │ 或输入"考考我"     │
                               ▼                   │
                    ┌──────────────────────┐       │
                    │    QUIZ (测验 Tab)    │       │ "继续学习"
                    │  Validator 出题评分   │───────┘
                    └──────────┬───────────┘
                               │ 测验完成 → 生成报告
                               ▼
                    ┌──────────────────────┐
                    │     COMPLETED        │
                    │  🎉 庆祝页 + 总结    │
                    └──────────┬───────────┘
                               │
                    ┌──────────┴───────────┐
                    ▼                      ▼
             "继续深入"              "新课程"
             → 回到 STUDY           → 回到 IDLE
```

### 8.2 Tab 切换与动作映射表

| 当前 Tab | 条件 | 用户动作 | 系统响应 |
|---------|------|---------|---------|
| Plan | 无计划 | 输入"生成计划"或点击按钮 | `handle_chat_input("请根据资料生成学习计划")` |
| Plan | 计划已生成 | 点击「学习」Tab | 切换到 Study Tab |
| Study | 学习中 | 输入"考考我"或点击「测验」Tab | `handle_generate_quiz()` |
| Quiz | 测验完成 | 输入"生成报告"或点击按钮 | `handle_generate_report()` |
| Quiz | 报告已生成 | 点击"查看总结" | 进入庆祝页 |

### 8.3 Tab 切换逻辑伪代码

```python
# src/ui/layout.py → _render_clickable_stepper()
# v0.3.0: 3 个可点击 Tab，点击直接切换

tab_keys = ["Plan", "Study", "Quiz"]
# 开发者模式下额外显示 Trace Tab
if dev_mode and show_trace:
    tab_keys.append("Trace")

# 每个 Tab 渲染为 HTML stepper-item，点击触发 st.experimental_rerun()
# 状态: active(橙色) / done(绿色✓) / default(灰色)
```
