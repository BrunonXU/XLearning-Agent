## 1. P0 验收 — 致命断路修复

### 1.1 Planner 真实输出（P0-1）

> 验证计划不是硬编码模板，而是根据输入内容动态生成

**测试 A：纯文本输入**
1. 首页输入框输入：`Python 异步编程`
2. 点击"开始学习" → 进入工作区
3. 点击引导横幅的"生成学习计划"按钮（或在聊天框输入"生成计划"）
4. 等待计划生成

- [ ] 计划中包含 `async`、`await`、`asyncio` 等 Python 异步相关术语
- [ ] 计划分为多个阶段（不是一段纯文字）
- [ ] 右侧面板显示计划结构化预览

**测试 B：PDF 输入**
1. 点击"新对话"回到首页
2. 上传一份 PDF（可用 `docs/test_cases/aloha.pdf` 或任意论文）
3. 等待 PDF 处理完成（聊天区应显示处理结果：页数、知识切片数）
4. 点击"生成学习计划"

- [ ] 计划中包含 PDF 内的关键术语（不是通用模板）
- [ ] 聊天区显示了 PDF 处理信息（页数、切片数）

**测试 C：GitHub URL 输入**
1. 点击"新对话"回到首页
2. 输入框输入：`https://github.com/langchain-ai/langchain`
3. 点击"开始学习" → "生成学习计划"

- [ ] 计划中包含 LangChain 相关术语（Agent、Chain、RAG 等）
- [ ] 系统识别出这是 GitHub URL 并调用了 RepoAnalyzer

## 2. P1 验收 — 全流程引导

## 5. Bug Fix 验收 (Day 7-8 修复项)

### 5.1 PDF 分析功能（BF-3/BF-5/BF-7）

> 验证上传 PDF 后，Tutor 能正确分析文档内容

1. 上传一份 PDF（如论文或技术文档）
2. 在聊天框输入：`分析`

- [ ] Tutor 回答的内容与 PDF 文档相关（不是返回 Python 示例）
- [ ] 不会出现"我没有看到 PDF"之类的回复
- [ ] 短输入"分析"被正确扩展为文档相关查询

3. 点击"生成学习计划"

- [ ] 计划标题使用 PDF 文档标题（不是"分析"）
- [ ] 计划内容与文档内容相关

### 5.2 跨 Session RAG 隔离（BF-2）

1. 创建 Session A，上传 PDF-A（如机器学习论文）
2. 聊天几轮，确认 RAG 正常工作
3. 点击"新对话"创建 Session B
4. 上传 PDF-B（如 Web 开发文档）
5. 在 Session B 中提问

- [ ] Session B 的回答基于 PDF-B（不会混入 PDF-A 的内容）

6. 切回 Session A

- [ ] Session A 的 RAG 上下文仍然正常

### 5.3 GitHub URL 识别（BF-9）

1. 首页输入：`https://github.com/psf/requests`

- [ ] 系统识别为 GitHub URL（不是当作普通文本）
- [ ] 调用 RepoAnalyzer 分析仓库

### 5.4 删除会话（BF-10）

1. 创建 2-3 个会话
2. 在侧边栏找到删除按钮，删除一个会话

- [ ] 会话从列表中消失
- [ ] 不影响其他会话

### 5.5 首次交互行为（BF-5）

1. 新建会话，不上传 PDF，输入"Python 入门"
2. 观察 Tutor 首次回复

- [ ] Tutor 询问学习目标或偏好（不是直接给出硬编码示例）

3. 新建会话，上传 PDF，输入"分析"

- [ ] Tutor 直接分析 PDF 内容（不再先问学习目标）
