# UI 截图回归指南

> 对应 TODO：UI-FIX-1 补充一组截图回归

在 UI 大改后，建议手动截取以下场景作为视觉回归参考，避免后续改版导致样式倒退。

## 建议截取场景

| 场景 | 操作步骤 | 验收点 |
|------|----------|--------|
| **首页** | 无 session 时打开应用 | 主输入框、6 个快捷示例、PDF expander 收起 |
| **工作区 - 短对话** | 输入主题 → 开始学习 → 进入工作区 | Stepper 吸顶、双列布局、分隔线、侧边栏 420px |
| **工作区 - 长对话** | 与 Agent 多轮对话至 5 条以上 | 聊天区可滚动，Stepper 仍吸顶可见 |
| **Plan 面板** | 生成计划后切到规划阶段 | 右侧计划预览、阶段列表、下载 .md 按钮 |
| **Study 面板** | 切到学习阶段 | 学习卡片/摘要区 |
| **庆祝页** | 完成全流程 → 查看总结 | 三指标卡片、出口按钮 |

## 存储建议

- 将截图放入 `docs/screenshots/` 并命名为 `01_home.png`、`02_workspace.png` 等
- 可加入 `.gitignore` 避免仓库体积膨胀，或单独建 `screenshots/` 分支
